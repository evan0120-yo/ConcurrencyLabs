# 商品 / 內容搜尋系統 · search

> 一句話定位：使用者輸入關鍵字找商品 / 內容，熱門查詢可快照，尖峰時要能降級保住可用。

---

## Block 1：這個情境同時符合哪些分類

- 業務 domain：社群 / Feed / 搜尋 / Marketplace
- 壓力模型：讀爆型（核心）

---

## Block 2：為什麼被列上這些

- **讀爆型** → 搜尋是重讀取（掃索引、算相關性、過濾排序），且熱門查詢高度集中。

---

## Block 3：User Story + 痛點分析（不涉及技術）

### User Story

```text
使用者搜「藍芽耳機」
   │
   ▼
[搜尋] 掃索引 ──> 算相關性 ──> 套篩選（價格 / 分類）──> 排序
   ├─ 熱門查詢（大促爆款詞）→ 反覆搜同一個 → 可快照
   └─ 尖峰扛不住 → 降級（少算點、給快取結果）

────────── 大促時大家都搜同幾個熱詞 ──────────

使用者想要：搜得到、夠相關、夠快
```

### 分段解說

- 搜尋是重讀取：不只查，還要算相關性與排序。
- 熱門查詢高度集中（大家搜同幾個詞），可快照結果。
- 尖峰扛不住時，寧可降級給「還可以的結果」也不要掛。

### 痛點分析

- **讀爆還寫爆？** 讀爆，且是重計算讀。
- **能不能快照 / 降級？** 熱門可快照，尖峰可降級。
- **有沒有熱點？** 熱門查詢詞是熱點。
- **真正的痛**：搜尋的重計算遇上熱詞集中與尖峰，要靠「熱門快照 + 可降級」在扛不住時仍給出可用結果。
