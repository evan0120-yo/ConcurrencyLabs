# 多支付通道路由 · paymentchannelrouting

> 一句話定位：一筆付款要從多個支付通道中選一個，通道會掛、會慢，要能健康度切換與 fallback。

---

## Block 1：這個情境同時符合哪些分類

- 業務 domain：支付 / 金融
- 壓力模型：平台防爆型（通道隔離 / fallback）、任務補償型（失敗切換重試）

---

## Block 2：為什麼被列上這些

- **平台防爆型** → 通道是外部依賴，會掛會慢，要靠健康度與隔離避免拖垮成交。
- **任務補償型** → 一個通道失敗要能切到備援重試。

---

## Block 3：User Story + 痛點分析（不涉及技術）

### User Story

```text
使用者要付 1000 元
   │
   ▼
[通道路由] 從多個通道挑一個
   ├─ 依 成本 / 成功率 / 國家 / 幣別 選
   ├─ (選到的通道很慢 / 掛了) ──> 切到備援
   └─ 熱門通道流量大 ──> 要限流保護

────────── 某通道半夜抽風，成功率驟降 ──────────

營運想要：自動避開壞掉的通道，別讓使用者付不了
```

### 分段解說

- 選通道不是固定規則，而是看即時健康度與成本。
- 通道會突然變差，路由要能感知並切換。
- 不同國家 / 幣別 / 商戶要能隔離，別互相影響。

### 痛點分析

- **不可控依賴** → 通道健康度隨時變。
- **要不要 fallback？** 要，壞通道要能自動繞開。
- **隔離** → 熱通道限流、國家/幣別/商戶分開。
- **真正的痛**：在一堆會抽風的外部通道上，動態選出「當下最可能成功」的那條，且壞掉能立刻切換。
