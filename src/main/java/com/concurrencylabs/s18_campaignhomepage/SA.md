# SA.md — s18_campaignhomepage（首頁活動頁）

> 事實文件（SASD）· 結構化分析。描述「首頁聚合要做什麼」的**業務行為**，平台無關（P1..Pn 共用；各平台怎麼扛量在各自 SD）。
> 特性：**聚合讀爆**。共用區塊對所有人相同（可預生成），個人化區塊因人而異（可延遲）；任一來源慢不得拖垮整頁。

## 1. Context Diagram

```text
[使用者] --(看首頁)--> (首頁聚合) --(共用主體秒開 + 個人化延遲補上)--> [使用者]

(首頁聚合) --(banner/商品列/排行 = 共用)--> [多來源 / 預生成快照]
(首頁聚合) --(個人化推薦)---------------> [推薦服務]（延遲載入）
```

## 2. DFD（精簡 box flow）

```text
[看首頁] --> [組頁]
   ├─【共用區塊】banner / 商品列 / 排行
   │     └─ 走預生成靜態快照（大家一樣，整份先生成）
   └─【個人化區塊】推薦
         └─ 延遲載入（主體先回，之後補）
   ▼
[聚合回應]（任一來源慢 / 掛 → 該區塊降級占位，不阻塞整頁）
```

## 3. Process Spec（行為基準，decision table）

```text
[共用區塊]
└─ 預生成整份快照，定時 / 事件更新；所有使用者讀同一份

[個人化區塊]
└─ 延遲載入（主體渲染後再補），慢一點可接受

[局部失敗容忍]（核心不變量）
├─ 某來源慢 → 逾時後該區塊顯示占位 / 舊快取，整頁照開
└─ 某來源掛 → 該區塊降級，不連累其他區塊與整頁開啟

邊界：整頁開啟時間由「最慢的核心區塊」決定 → 核心區塊一律走快取 / 預生成，不即時回源。
```

## 4. State Transition

```text
（讀取聚合型，無業務狀態機；只有預生成快照的 生成 / 更新 / 失效 生命週期）
```

## 5. Data Dictionary（邏輯欄位；實體 DDL 在各平台 SD）

```text
page_snapshot（首頁預生成快照）
├─ pageId(PK) / version / blocks[]（共用區塊渲染結果）
└─ generatedAt / ttl

block（區塊定義）
├─ blockId / type（SHARED|PERSONALIZED）/ source（來源服務）
└─ degradePolicy（占位 / 舊快取 / 隱藏）
```
