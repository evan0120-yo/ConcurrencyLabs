# 彩金獎池累積系統 · s03_jackpotpool

> 一句話定位：每筆下注抽成累加進獎池，數字不斷跳動放大，有人中大獎時一次結算歸零。

---

## Block 1：這個情境同時符合哪些分類

- 業務 domain：博弈 / 遊戲
- 壓力模型：熱點計數型（核心）、錢包帳本型（中獎結算）、讀爆型（大家盯著看池）

---

## Block 2：為什麼被列上這些

- **熱點計數型** → 成千上萬筆下注同時累加「同一個獎池數字」，是最典型的單點熱 key。
- **錢包帳本型** → 中獎那一刻要把池金額精準派給玩家，涉及真錢、不可錯。
- **讀爆型** → 前端獎池數字不斷跳動，被大量玩家高頻讀取。

---

## Block 3：User Story + 痛點分析（不涉及技術）

### User Story

```text
【累積】每筆下注抽 1% 進獎池
   玩家 A 下注 ──> 池 +X
   玩家 B 下注 ──> 池 +Y      （上萬筆同時累加同一個池）
   ▼
[獎池數字] 前端不斷跳動放大 ← 所有人一直盯著看

────────── 有人中大獎 ──────────

   └─(中獎)──> 一次結算派彩 ──> 池歸零、重新累積
```

### 分段解說

- 平常：壓力是「同一個數字被高頻累加」＋「同一個數字被高頻讀」。
- 中獎：性質瞬間切換成「精準結算一大筆錢」。
- 顯示與結算是兩種一致性：看的可以近似，發的必須精準。

### 痛點分析

- **有沒有熱點？** 有，且是最極端的單點熱 key。
- **能不能近似？** 顯示數字可近似跳動；中獎結算金額不可近似。
- **讀爆還寫爆？** 讀寫都兇（累加寫 + 圍觀讀）。
- **真正的痛**：同一個數字既要承受高頻累加與圍觀，又要在中獎瞬間切換成強一致結算。
