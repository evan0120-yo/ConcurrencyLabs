# SA.md — s20_shoppingcart（購物車系統）

> 事實文件（SASD）· 結構化分析。描述「購物車要做什麼」的**業務行為**，平台無關（P1..Pn 共用；各平台怎麼扛量在各自 SD）。
> 特性：高頻加/改/刪、跨裝置同步，**展示可弱一致**；唯一嚴格的點是「結帳前一刻」要用最新價格/庫存/上下架校正。

## 1. Context Diagram

```text
[使用者多端] --(加/改/刪購物車: userId, itemId, qty)--> (購物車) --(車內容)--> [使用者]
[結帳前]     --(校正: userId)---------------------------> (購物車) --(校正後可結帳清單)--> [使用者]

(購物車) --(車內容讀寫)--> [PostgreSQL cart + util.redis cache]
(購物車) --(結帳校正: 抓最新價/庫存/上下架)--> [s19 價格 / s15 庫存 / 商品]
```

## 2. DFD（精簡 box flow）

```text
【日常（弱一致）】
[加/改/刪] --> [寫車]（短時間合併寫入）--> [DB + cache]
[看車] --> 走 cache（可稍舊、跨裝置晚幾秒同步 OK）

【結帳前（強校正）】
[結帳校正] --(逐項)-->
   ├─ 商品下架 → 標記失效 / 移除
   ├─ 價格變動 → 更新為最新價
   └─ 庫存不足 → 標記不可購 / 調整數量
   ▼
回應「校正後可結帳清單」
```

## 3. Process Spec（行為基準，decision table）

```text
[日常讀寫]
├─ 展示允許弱一致（cache 稍舊可接受）
└─ 跨裝置多端寫入 → 最終一致收斂（晚幾秒 OK）

[高頻小寫]
└─ 加減數量等高頻小寫入可短時合併，降低寫放大

[結帳前校正]（唯一強一致點）
├─ 下架 → 移除 / 標失效
├─ 價格 → 抓 s19 最新價（不得用舊價）
└─ 庫存 → 對 s15 校驗（不得賣缺貨）

不變量：平常放鬆換效能，但「結帳前一刻」必須以最新真相把關。
```

## 4. State Transition（一個購物車項）

```text
【IN_CART】──(結帳校正·正常)──> 【CHECKABLE】
        ├──(下架)──> 【INVALID】（移除）
        └──(缺貨)──> 【OUT_OF_STOCK】（不可購 / 調量）
```

## 5. Data Dictionary（邏輯欄位；實體 DDL 在各平台 SD）

```text
cart（購物車）
├─ userId(PK) / updatedAt
└─ items[]

cart_item（車項）
├─ itemId / qty / addedAt
└─ snapshotPrice（展示用，結帳前以最新價覆蓋）
```
