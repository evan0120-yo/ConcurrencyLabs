# SA.md — s58_search（商品 / 內容搜尋系統）

> 事實文件（SASD）· 結構化分析。描述「搜尋要做什麼」的**業務行為**，平台無關（P1..Pn 共用；各平台怎麼扛量在各自 SD）。
> 特性：**重讀取**（掃索引、算相關性、過濾排序），熱門查詢高度集中。靠「熱門快照 + 可降級」在扛不住時仍給出可用結果。

## 1. Context Diagram

```text
[使用者] --(搜尋: keyword, 篩選, 排序)--> (搜尋) --(結果列表)--> [使用者]

(搜尋) --(掃索引 / 相關性)--> [搜尋索引 ES]
(搜尋) --(熱門查詢快照)------> [util.redis]
```

## 2. DFD（精簡 box flow）

```text
[搜尋] --(keyword, filters, sort)--> [查快照]
   │(熱門查詢命中)──> 直接回快照結果（省重算）
   │(未命中)
   ▼
[掃索引] --> [算相關性] --> [套篩選(價格/分類)] --> [排序]
   ▼
[寫快照]（熱門查詢：快取結果）── 回結果
   │
   （尖峰扛不住時）──> [降級]：少算相關性 / 給快取 / 縮結果集，保住可用
```

## 3. Process Spec（行為基準，decision table）

```text
[熱門快照]
├─ 熱門查詢（大促爆款詞，大家搜同幾個）→ 快取結果，命中直接回
└─ 一般查詢 → 走索引重算

[搜尋流程]
└─ 掃索引 → 相關性打分 → 篩選 → 排序（重讀取）

[降級]（尖峰保可用，核心）
├─ 扛不住 → 少算相關性 / 給稍舊快取 / 縮小結果集
└─ 寧可給「還可以的結果」也不要掛（可用性優先）

[一致性]
└─ 結果可稍舊（索引同步有延遲）；搜尋非強一致場景
```

## 4. State Transition

```text
（查詢型，無業務狀態機；查詢的 命中快照 / 索引重算 / 降級 路徑）
```

## 5. Data Dictionary（邏輯欄位；實體 DDL 在各平台 SD）

```text
search_index（搜尋索引，通常 ES）
├─ docId / fields（title, category, price, …）/ 分詞 / 權重

query_snapshot（熱門查詢快照，活在 Redis）
├─ queryKey（keyword+filters+sort）/ results[] / cachedAt
```
