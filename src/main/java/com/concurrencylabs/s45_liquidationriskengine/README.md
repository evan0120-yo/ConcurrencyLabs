# 強平與風控引擎 · s45_liquidationriskengine

> 一句話定位：即時盯著所有槓桿倉位與價格，跌破維持保證金就強制平倉，行情暴動時是生死時刻。

---

## Block 1：這個情境同時符合哪些分類

- 業務 domain：交易所 / Web3
- 壓力模型：低延遲型（價格驅動即時檢查）、熱點型（暴動時集中觸發）

---

## Block 2：為什麼被列上這些

- **低延遲型** → 價格一動就要重算風險，慢了就穿倉、平台承損。
- **熱點型** → 行情暴動時同一 symbol 大量倉位同時逼近強平線。

---

## Block 3：User Story + 痛點分析（不涉及技術）

### User Story

```text
價格每一跳動
   │
   ▼
[強平引擎] 重新檢查：哪些倉位跌破維持保證金？
   ├─ (安全) ──> 略過
   └─ (危險) ──> 觸發強制平倉

────────── 行情瞬間暴跌 20% ──────────

大量倉位同時逼近爆倉線
   ├─ 引擎想要：即時算完、該平的即時平
   └─ 平台風險：算慢一步 = 穿倉 = 平台吃虧
```

### 分段解說

- 強平是「價格事件驅動」：價格動就要重算。
- 暴動時大量倉位同時危險，是最需要算力的時刻。
- 強平引擎要獨立資源、風控優先，不能被別的流量搶算力。

### 痛點分析

- **延遲要求** → 極低，慢一步就穿倉。
- **有沒有熱點？** 暴動時大量倉位集中觸發。
- **要不要隔離？** 要，風控算力優先於一般功能。
- **真正的痛**：最需要算力的時刻（暴動）正是所有倉位一起危險的時刻，引擎必須在洪峰下仍即時算完。

---

## 容量平台演進地圖（P1 → P4）

> 起點固定假設 QPS 3,000+（量體為價格驅動的倉位重算）。每跳一個平台 = 一次架構改造。
> s45 極低延遲 + 暴動洪峰(所有倉位一起危險) + 算力優先 → 共 4 個平台。

```text
【P1 · 基線】約 3,000 QPS
├─ 定義：價格事件驅動,逐倉重算保證金率,跌破觸發強平
├─ 大方塊：價格驅動重算 · 保證金率判定 · 觸發市價平倉
└─ 撐不住的訊號 → 暴動時大量倉位同時逼近強平線、全量重算算力不足

【P2 · 增量計算 + 風控獨立資源】約 10,000 QPS
├─ 定義：價格變動只重算受影響倉位(增量),風控引擎資源獨立優先
├─ 大方塊（綁一起）
│   ├─ 增量計算(只算受該價格影響的倉位)
│   ├─ 風控算力獨立(不與一般流量搶)
│   └─ 倉位索引(按 symbol/價格帶快速定位危險倉)
├─ 為何綁：即時算完 = 增量計算 + 獨立資源 + 快速定位,一起做
└─ 撐不住的訊號 → 單 symbol 倉位量巨大、暴動集中觸發

【P3 · 按 symbol 分片並行風控】約 50,000 QPS
├─ 定義：風控按 symbol 分片並行,各片獨立重算與觸發
├─ 大方塊：symbol 分片 · 各片獨立風控引擎 · 分片內強平佇列
└─ 為何綁：一分片,倉位重算/觸發/結算收斂到該 symbol

【P4 · 暴動洪峰預案 + 獨立高規部署 + ADL 兜底】約 100,000+ QPS
├─ 定義：暴動洪峰下獨立高規部署,穿倉走風險準備金/ADL,優先級最高
├─ 大方塊：獨立高規部署 · 洪峰降級預案 · 穿倉 ADL 兜底
└─ 為何綁：生死時刻不崩 = 高規部署 + 洪峰預案 + ADL,一起做
```
