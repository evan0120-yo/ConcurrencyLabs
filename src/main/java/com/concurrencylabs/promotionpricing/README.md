# 促銷價格計算 · promotionpricing

> 一句話定位：每次看到 / 結帳的價格是疊加各種活動規則算出來的，熱門商品可快照、活動期間要凍結一致。

---

## Block 1：這個情境同時符合哪些分類

- 業務 domain：電商 / 促銷
- 壓力模型：讀爆型（熱門商品被反覆算）

---

## Block 2：為什麼被列上這些

- **讀爆型** → 熱門商品的價格被大量、反覆計算與查詢，可用快照擋掉重複計算。

---

## Block 3：User Story + 痛點分析（不涉及技術）

### User Story

```text
每次看到 / 結帳的價格 = 疊加多種規則
   商品 ──> [算價] 疊：滿減 + 券 + 會員價 + 限時折 ──> 最終價
   ├─ 熱門商品 → 一直被算 ──> 可快照
   └─ 活動期間 → 價格要凍結一致（不能一刷一個價）

────────── 規則越疊越多，算一次成本越來越高 ──────────
```

### 分段解說

- 價格不是欄位，是「一串規則疊出來的結果」。
- 熱門商品被反覆用相同輸入算相同結果，浪費算力，可快照。
- 活動期間同一商品價格要穩定，不能因規則刷新忽高忽低。

### 痛點分析

- **讀爆還寫爆？** 讀爆（大量算價 = 大量讀規則）。
- **能不能快照？** 熱門商品可快照，省重複計算。
- **一致性** → 活動期間價格要凍結，避免抖動。
- **真正的痛**：算價規則會越疊越複雜，如何在「規則正確」與「別每次都重算」之間取得平衡。
