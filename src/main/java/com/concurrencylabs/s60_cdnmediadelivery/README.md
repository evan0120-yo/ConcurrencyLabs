# 媒體內容分發 / CDN 整合 · s60_cdnmediadelivery

> 一句話定位：把圖片 / 影片就近分發給使用者，熱門內容推到邊緣，保護原站不被打爆。

---

## Block 1：這個情境同時符合哪些分類

- 業務 domain：社群 / Feed / 搜尋 / Marketplace
- 壓力模型：讀爆型（核心）

---

## Block 2：為什麼被列上這些

- **讀爆型** → 媒體檔被海量使用者重複讀取，是頻寬密集型讀爆。

---

## Block 3：User Story + 痛點分析（不涉及技術）

### User Story

```text
使用者滑到一張圖 / 一支影片
   │
   ▼
[內容分發]
   ├─ 熱門內容 → 已推到離使用者最近的邊緣節點 → 就近拿、超快
   └─ 冷門內容 → 邊緣沒有 → 回原站拿一次，再快取起來
   *目標：絕大多數請求都不要打到原站*

────────── 一支影片爆紅，全國同時看 ──────────

原站想要：別被熱門內容的流量直接打爆
```

### 分段解說

- 媒體分發的核心是「就近」與「保護原站」。
- 熱門內容推到邊緣，讓使用者就近拿。
- 冷門內容第一次回原站，之後也被快取。

### 痛點分析

- **讀爆還寫爆？** 讀爆，且是頻寬密集。
- **有沒有熱點？** 爆紅內容是熱點。
- **原站保護** → 絕大多數請求不該回原站。
- **真正的痛**：熱門媒體的流量若直打原站會瞬間爆掉，要靠邊緣快取把原站擋在後面、只承受少量回源。

---

## 容量平台演進地圖（P1 → P3）

> 起點固定假設 QPS 3,000+。每跳一個平台 = 一次架構改造（一組必須一起改的大方塊）。
> s60 頻寬密集讀爆、邊緣快取保護原站,放寬 → 共 3 個平台。

```text
【P1 · 基線】約 3,000 QPS
├─ 定義：原站供媒體 + 基本快取
├─ 大方塊：原站儲存 · 快取
└─ 撐不住的訊號 → 爆紅內容直打原站瞬間爆、頻寬密集讀

【P2 · CDN 邊緣分發 + 熱門推邊緣 + 回源保護】約 10,000 QPS
├─ 定義：靜態媒體上 CDN 邊緣,熱門主動預熱,回源單飛防擊穿
├─ 大方塊（綁一起）
│   ├─ CDN 邊緣就近分發
│   ├─ 熱門內容主動預熱到邊緣
│   └─ 回源單飛(同冷內容只回源一次)
├─ 為何綁：保護原站 = 邊緣分發 + 預熱 + 回源單飛,一起做
└─ 撐不住的訊號 → 超級爆紅內容、邊緣命中率仍不足

【P3 · 多層邊緣 + 熱點預熱 + 分段快取】約 50,000+ QPS
├─ 定義：多層邊緣(邊緣→區域→原站),熱點智慧預熱,大檔分段快取
├─ 大方塊：多層邊緣 · 熱點智慧預熱 · 大檔分段快取
└─ 為何綁：頻寬密集讀爆 = 多層邊緣 + 預熱 + 分段,一起做
```
