# SA.md — s60_cdnmediadelivery（媒體內容分發 / CDN 整合）

> 事實文件（SASD）· 結構化分析。描述「媒體分發要做什麼」的**業務行為**，平台無關（P1..Pn 共用；各平台怎麼扛量在各自 SD）。
> 核心是**就近**與**保護原站**：熱門內容推到邊緣讓使用者就近拿，絕大多數請求都不該回原站；爆紅內容的流量若直打原站會瞬間爆掉。

## 1. Context Diagram

```text
[使用者] --(取媒體: mediaId)--> (內容分發) --(就近回內容)--> [使用者]

(內容分發) --(邊緣命中 / 回源)--> [CDN 邊緣節點]
(內容分發) --(回源一次)--------> [原站 object storage]
```

## 2. DFD（精簡 box flow）

```text
[取媒體] --(mediaId)--> [就近邊緣節點]
   ├─(邊緣命中)──> 直接回（超快、就近，不打原站）
   │(邊緣未命中)
   ▼
[回源]（向原站取一次）
   ├─ 取得 → 回使用者 + 快取到邊緣（之後就命中）
   └─ 防擊穿：同一冷內容大量並發回源 → 單飛（只回源一次）
   ▼
（熱門內容）──> 主動預熱推到邊緣（爆紅前/後）
```

## 3. Process Spec（行為基準，decision table）

```text
[就近命中]
├─ 邊緣有 → 就近回（絕大多數請求）
└─ 邊緣無 → 回源一次後快取

[原站保護]（核心不變量）
├─ 目標：絕大多數請求不回原站
├─ 冷內容首次回源後即快取；同一內容並發回源單飛（防擊穿）
└─ 熱門 / 爆紅內容主動預熱到邊緣，避免流量直打原站

[快取策略]
└─ TTL / 版本化（內容更新 → 換 URL / 主動刷新）；大檔分段快取
```

## 4. State Transition（一個媒體物件的邊緣狀態）

```text
【NOT_CACHED】──(首次回源)──> 【CACHED（邊緣）】──(TTL 到 / 更新)──> 失效 → 重新回源
      熱門 → 主動 PRELOADED（預熱）
```

## 5. Data Dictionary（邏輯欄位；實體 DDL 在各平台 SD）

```text
media_object（媒體物件）
├─ mediaId(PK) / originUrl / version / contentType / size

edge_cache_meta（邊緣快取中繼，觀測 / 預熱用）
├─ mediaId / edgeNode / cachedAt / ttl / hot（是否預熱）
```
